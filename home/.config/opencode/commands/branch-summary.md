@git-branch-diff

Draft a PR description from the branch changes above following the repository's conventions.

## Instructions

1. Analyze the commits and changes to understand the purpose and scope
2. Start with the "why" before the "what"
3. Highlight breaking changes prominently
4. Keep it scannable with bullet points
5. Mention any dependencies or related PRs

## Output Format

```markdown
## Summary

Brief description of what this PR does and why.

## Changes

- List of significant changes made

## Testing

How to test these changes, if applicable.

## Notes

Any additional context, breaking changes, or migration steps.
```
